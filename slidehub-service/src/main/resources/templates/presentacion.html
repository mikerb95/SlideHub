SlideHub
Sistema de Presentación Multi-Pantalla en Tiempo Real
Java 21 Spring Boot 4 Microservicios IA Generativa OAuth2 Redis
Objetivo General
Reescribir y escalar el módulo de presentaciones
Desarrollar una plataforma distribuida de control de presentaciones en tiempo real, migrando el módulo monolítico PHP/CodeIgniter 4 a una arquitectura de microservicios Java con Spring Boot, integrando inteligencia artificial generativa para asistir al presentador y automatizar el despliegue.
⚙
De monolito a microservicios
4 servicios independientes comunicados por REST API a través de un gateway centralizado.
★
IA como valor agregado
Gemini Vision + Groq generan notas del presentador y guías de despliegue automáticas.
☁
Cloud-native
Desplegado en Render con Redis, PostgreSQL (Aiven), MongoDB Atlas y S3.
Objetivos Específicos
Metas concretas del proyecto
  1. 1 Diseñar e implementar 4 microservicios (Gateway, State, UI, AI) con responsabilidades separadas y comunicación REST.
  2. 2 Implementar sincronización en tiempo real de diapositivas entre N pantallas usando Redis como store de estado.
  3. 3 Integrar autenticación local (BCrypt) y OAuth2 (GitHub + Google) con Spring Security y persistencia en PostgreSQL.
  4. 4 Desarrollar un módulo de IA que genere notas del presentador mediante Gemini Vision y Groq, almacenadas en MongoDB.
  5. 5 Construir un "Deploy Tutor" que analice repositorios y genere Dockerfiles + guías de despliegue automáticas.
  6. 6 Implementar importación de diapositivas desde Google Drive y subida manual a Amazon S3.
  7. 7 Desplegar toda la plataforma en Render con integración continua y configuración por perfiles (dev/prod).

ISO/IEC 27001 Seguridad de la Información
Controles de seguridad implementados
ISO 27001 establece un marco para el Sistema de Gestión de Seguridad de la Información (SGSI). SlideHub aplica controles del Anexo A relevantes para una aplicación web distribuida.
🔒
A.9 — Control de Acceso
Spring Security con roles (HOST, PRESENTER). Rutas protegidas por rol. Sesiones invalidadas en logout. OAuth2 como segundo factor de identidad.
🔐
A.10 — Criptografía
Contraseñas hasheadas con BCrypt (factor 10). Tokens OAuth2 encriptados en PostgreSQL. SSL obligatorio en Aiven (PostgreSQL). HTTPS en producción.
🖥
A.12 — Seguridad Operacional
Variables de entorno para secretos (nunca hardcoded). Perfiles separados dev/prod. Health checks y actuator para monitoreo. Logs con SLF4J estructurado.
🌐
A.13 — Seguridad en Comunicaciones
Gateway centralizado como único punto de entrada. CORS y headers configurados en gateway-service. Comunicación interna REST entre servicios en red privada.
💻
A.14 — Desarrollo Seguro
Validación de inputs en boundaries. Flyway para migraciones controladas (nunca ddl-auto=create). Inyección por constructor (no reflexión). Sin SDKs innecesarios.
📦
A.15 — Relaciones con Proveedores
APIs externas (Gemini, Groq, Resend) accedidas solo vía WebClient HTTP — sin SDKs que inyecten dependencias opacas. AWS SDK v2 como excepción justificada (SigV4).
